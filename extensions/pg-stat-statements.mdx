---
title: pg_stat_statements
description: Track planning and execution statistics for all SQL statements executed by a PostgreSQL server.
---

The `pg_stat_statements` module tracks planning and execution statistics for every distinct SQL statement executed by the server. It is the primary tool for identifying slow queries, high-frequency queries, and I/O-intensive statements.

## Installation

<Steps>
  <Step title="Add to shared_preload_libraries">
Edit `postgresql.conf` and add `pg_stat_statements` to `shared_preload_libraries`:

```ini
# postgresql.conf
shared_preload_libraries = 'pg_stat_statements'
```

`pg_stat_statements` requires additional shared memory and must be loaded at server start. A restart is required after changing this parameter.
  </Step>
  <Step title="Restart PostgreSQL">
```bash
pg_ctl restart -D /var/lib/postgresql/17/main
# or
systemctl restart postgresql
```
  </Step>
  <Step title="Create the extension">
Connect to the database where you want to use the extension and run:

```sql
CREATE EXTENSION pg_stat_statements;
```

This creates the `pg_stat_statements` view and related functions in the current database.
  </Step>
</Steps>

<Note>
Query identifier calculation must be enabled for the module to collect data. This happens automatically when `compute_query_id` is set to `auto` (the default) or `on`.
</Note>

## Configuration parameters

Set these in `postgresql.conf`:

| Parameter | Default | Description |
|-----------|---------|-------------|
| `pg_stat_statements.max` | 5000 | Maximum number of distinct statements to track. When exceeded, least-executed statements are evicted. |
| `pg_stat_statements.track` | `top` | Which statements to track: `top` (top-level only), `all` (including statements inside functions), or `none`. |
| `pg_stat_statements.track_planning` | `off` | Track planning statistics in addition to execution statistics. |
| `pg_stat_statements.track_utility` | `on` | Track utility statements (non-SELECT/INSERT/UPDATE/DELETE). |
| `pg_stat_statements.save` | `on` | Save statistics across server restarts. |

Example:

```ini
# postgresql.conf
shared_preload_libraries = 'pg_stat_statements'
pg_stat_statements.max = 10000
pg_stat_statements.track = all
pg_stat_statements.track_planning = on
track_io_timing = on   # enables blk_read_time and blk_write_time
```

## The pg_stat_statements view

The view contains one row for each distinct combination of database, user, query ID, and whether it is a top-level statement.

### Key columns

| Column | Type | Description |
|--------|------|-------------|
| `userid` | oid | OID of the user who executed the statement |
| `dbid` | oid | OID of the database in which the statement was executed |
| `queryid` | bigint | Hash code identifying the normalized query |
| `query` | text | Text of a representative statement (with literals replaced by `$1`, `$2`, etc.) |
| `calls` | bigint | Number of times the statement was executed |
| `total_exec_time` | double precision | Total time spent executing the statement, in milliseconds |
| `min_exec_time` | double precision | Minimum execution time in milliseconds |
| `max_exec_time` | double precision | Maximum execution time in milliseconds |
| `mean_exec_time` | double precision | Mean execution time in milliseconds |
| `stddev_exec_time` | double precision | Population standard deviation of execution time in milliseconds |
| `rows` | bigint | Total rows retrieved or affected |
| `shared_blks_hit` | bigint | Total shared block cache hits |
| `shared_blks_read` | bigint | Total shared blocks read from disk |
| `shared_blks_dirtied` | bigint | Total shared blocks dirtied |
| `shared_blks_written` | bigint | Total shared blocks written |
| `local_blks_hit` | bigint | Total local block cache hits (temp tables) |
| `local_blks_read` | bigint | Total local blocks read |
| `shared_blk_read_time` | double precision | Time spent reading shared blocks (ms, if `track_io_timing` is on) |
| `shared_blk_write_time` | double precision | Time spent writing shared blocks (ms, if `track_io_timing` is on) |
| `temp_blks_read` | bigint | Total temp file blocks read |
| `temp_blks_written` | bigint | Total temp file blocks written |
| `wal_records` | bigint | Total WAL records generated |
| `wal_bytes` | numeric | Total WAL bytes generated |
| `plans` | bigint | Number of times the statement was planned (requires `track_planning = on`) |
| `total_plan_time` | double precision | Total time spent planning, in milliseconds |
| `mean_plan_time` | double precision | Mean planning time in milliseconds |
| `stats_since` | timestamptz | When statistics collection started for this statement |
| `toplevel` | bool | True if executed as a top-level statement |
| `parallel_workers_launched` | bigint | Parallel workers actually launched |
| `generic_plan_calls` | bigint | Executions using a generic plan |
| `custom_plan_calls` | bigint | Executions using a custom plan |

## Useful queries

<AccordionGroup>
  <Accordion title="Slowest queries by mean execution time">
```sql
SELECT
  round(mean_exec_time::numeric, 2)  AS mean_ms,
  round(total_exec_time::numeric, 2) AS total_ms,
  calls,
  round(stddev_exec_time::numeric, 2) AS stddev_ms,
  rows,
  query
FROM pg_stat_statements
ORDER BY mean_exec_time DESC
LIMIT 20;
```
  </Accordion>
  <Accordion title="Most time-consuming queries overall">
```sql
SELECT
  round(total_exec_time::numeric, 2) AS total_ms,
  calls,
  round(mean_exec_time::numeric, 2)  AS mean_ms,
  round(
    100.0 * total_exec_time /
    sum(total_exec_time) OVER (),
    2
  ) AS pct_total,
  query
FROM pg_stat_statements
ORDER BY total_exec_time DESC
LIMIT 20;
```
  </Accordion>
  <Accordion title="Highest I/O queries (shared block reads from disk)">
```sql
SELECT
  shared_blks_read,
  shared_blks_hit,
  round(
    100.0 * shared_blks_hit /
    nullif(shared_blks_hit + shared_blks_read, 0),
    2
  ) AS cache_hit_pct,
  calls,
  query
FROM pg_stat_statements
ORDER BY shared_blks_read DESC
LIMIT 20;
```
  </Accordion>
  <Accordion title="Queries that generate the most WAL">
```sql
SELECT
  wal_bytes,
  wal_records,
  calls,
  round(mean_exec_time::numeric, 2) AS mean_ms,
  query
FROM pg_stat_statements
ORDER BY wal_bytes DESC
LIMIT 20;
```
  </Accordion>
  <Accordion title="Queries with high execution time variance (unstable)">
```sql
SELECT
  round(mean_exec_time::numeric, 2)   AS mean_ms,
  round(stddev_exec_time::numeric, 2) AS stddev_ms,
  round(max_exec_time::numeric, 2)    AS max_ms,
  calls,
  query
FROM pg_stat_statements
WHERE calls > 10
ORDER BY stddev_exec_time DESC
LIMIT 20;
```
  </Accordion>
  <Accordion title="Queries with slow block I/O (requires track_io_timing = on)">
```sql
SELECT
  round(shared_blk_read_time::numeric, 2)  AS read_ms,
  round(shared_blk_write_time::numeric, 2) AS write_ms,
  shared_blks_read,
  calls,
  query
FROM pg_stat_statements
WHERE shared_blk_read_time > 0
ORDER BY shared_blk_read_time DESC
LIMIT 20;
```
  </Accordion>
  <Accordion title="Most frequently called queries">
```sql
SELECT
  calls,
  round(mean_exec_time::numeric, 2) AS mean_ms,
  round(total_exec_time::numeric, 2) AS total_ms,
  query
FROM pg_stat_statements
ORDER BY calls DESC
LIMIT 20;
```
  </Accordion>
</AccordionGroup>

## Resetting statistics

```sql
-- Reset statistics for all statements in all databases
SELECT pg_stat_statements_reset();

-- Reset statistics for a specific user and database
SELECT pg_stat_statements_reset(
  userid := (SELECT oid FROM pg_roles WHERE rolname = 'alice'),
  dbid   := (SELECT oid FROM pg_database WHERE datname = 'mydb')
);

-- Reset only min/max statistics (not counts or totals)
SELECT pg_stat_statements_reset(minmax_only := true);
```

## The pg_stat_statements_info view

This view provides metadata about the extension itself:

```sql
SELECT * FROM pg_stat_statements_info;
```

| Column | Description |
|--------|-------------|
| `deallocs` | Number of times the least-executed entry was evicted due to `pg_stat_statements.max` being reached |
| `stats_reset` | Time of the last statistics reset |

A high `deallocs` value means you are tracking more distinct queries than `pg_stat_statements.max` allows. Increase the limit:

```ini
# postgresql.conf
pg_stat_statements.max = 20000
```
