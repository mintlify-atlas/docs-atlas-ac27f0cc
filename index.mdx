## Why PostgreSQL?

PostgreSQL is not just a database — it's an extensible platform that goes far beyond standard SQL.

<CardGroup cols={3}>
  <Card title="ACID Transactions" icon="shield-check">
    Full ACID compliance with multi-version concurrency control (MVCC) ensures data integrity without sacrificing performance.
  </Card>
  <Card title="Advanced SQL" icon="code">
    Window functions, CTEs, lateral joins, full-text search, and an extended SQL standard that keeps growing.
  </Card>
  <Card title="JSON & Document Store" icon="brackets-curly">
    Native JSON and JSONB types let you store and query document data alongside relational data in the same database.
  </Card>
  <Card title="Extensibility" icon="puzzle-piece">
    Define custom types, functions, operators, index methods, and procedural languages. Build extensions that integrate deeply with the engine.
  </Card>
  <Card title="High Availability" icon="server">
    Streaming replication, logical replication, and built-in support for standby servers and failover.
  </Card>
  <Card title="Security" icon="lock">
    Row-level security, SSL/TLS, SCRAM-SHA-256 authentication, and fine-grained privilege management.
  </Card>
</CardGroup>

## About this documentation

This documentation covers PostgreSQL's source code structure, SQL dialect, administration, developer interfaces, and extension APIs. It is generated from the [postgres/postgres](https://github.com/postgres/postgres) mirror of the official PostgreSQL repository.

<Note>
  PostgreSQL development happens on the official [PostgreSQL mailing lists](https://www.postgresql.org/community/lists/). Pull requests on GitHub are not reviewed — to contribute, see the [Submitting a Patch](https://wiki.postgresql.org/wiki/Submitting_a_Patch) wiki page.
</Note>
