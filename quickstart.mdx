This guide walks you through installing PostgreSQL, creating a database cluster, connecting with `psql`, and running your first SQL queries.

## Install PostgreSQL

<Tabs>
  <Tab title="Ubuntu / Debian">
    ```bash
    # Install from the official PostgreSQL apt repository
    sudo apt install -y postgresql-common
    sudo /usr/share/postgresql-common/pgdg/apt.postgresql.org.sh

    # Install PostgreSQL 17 (or the latest available version)
    sudo apt install -y postgresql-17
    ```

    The package manager automatically creates the `postgres` system user and initializes a database cluster in `/var/lib/postgresql/17/main`.
  </Tab>
  <Tab title="RHEL / Fedora">
    ```bash
    # Install the repository RPM
    sudo dnf install -y https://download.postgresql.org/pub/repos/yum/reporpms/EL-9-x86_64/pgdg-redhat-repo-latest.noarch.rpm

    # Disable the built-in PostgreSQL module
    sudo dnf -qy module disable postgresql

    # Install PostgreSQL 17
    sudo dnf install -y postgresql17-server

    # Initialize the cluster and enable on boot
    sudo /usr/pgsql-17/bin/postgresql-17-setup initdb
    sudo systemctl enable --now postgresql-17
    ```
  </Tab>
  <Tab title="macOS (Homebrew)">
    ```bash
    brew install postgresql@17

    # Start the service
    brew services start postgresql@17

    # Add to PATH (add to your shell profile to make permanent)
    export PATH="/opt/homebrew/opt/postgresql@17/bin:$PATH"
    ```
  </Tab>
  <Tab title="Windows">
    Download the interactive installer from [postgresql.org/download/windows](https://www.postgresql.org/download/windows/). The installer sets up the server, initializes a cluster, and optionally installs pgAdmin and Stack Builder.
  </Tab>
</Tabs>

## Connect to PostgreSQL

<Steps>
  <Step title="Switch to the postgres user">
    On Linux, the PostgreSQL server runs as the `postgres` operating system user. Use `sudo` to become that user:

    ```bash
    sudo -u postgres psql
    ```

    On macOS with Homebrew, you connect as your own user:

    ```bash
    psql postgres
    ```

    You should see the `psql` prompt:

    ```
    psql (17.0)
    Type "help" for help.

    postgres=#
    ```
  </Step>

  <Step title="Create a database and user">
    ```sql
    -- Create a dedicated database user
    CREATE USER myuser WITH PASSWORD 'mypassword';

    -- Create a database owned by that user
    CREATE DATABASE mydb OWNER myuser;

    -- List databases to confirm
    \l
    ```
  </Step>

  <Step title="Connect to the new database">
    ```sql
    -- Connect to the new database within psql
    \c mydb myuser
    ```

    Or connect directly from the shell:

    ```bash
    psql -U myuser -d mydb -h localhost
    ```
  </Step>
</Steps>

## Create a table and insert data

The classic PostgreSQL tutorial uses a weather and cities example from the source at `src/tutorial/`:

```sql
-- Create the tables
CREATE TABLE cities (
    name            varchar(80),
    location        point
);

CREATE TABLE weather (
    city            varchar(80) references cities(name),
    temp_lo         int,           -- low temperature
    temp_hi         int,           -- high temperature
    prcp            real,          -- precipitation
    date            date
);

-- Insert some data
INSERT INTO cities VALUES ('San Francisco', '(-194.0, 53.0)');

INSERT INTO weather VALUES
    ('San Francisco', 46, 50, 0.25, '1994-11-27'),
    ('San Francisco', 43, 57, 0.0,  '1994-11-29'),
    ('Hayward',       37, 54, NULL, '1994-11-29');
```

## Run your first queries

```sql
-- Retrieve all weather records
SELECT * FROM weather;

-- Join weather with cities
SELECT city, temp_lo, temp_hi, prcp, date, location
FROM weather JOIN cities ON city = name;

-- Aggregate: find the highest low-temperature per city
SELECT city, max(temp_lo) AS max_low
FROM weather
GROUP BY city;
```

<Tip>
  Use `\x` in psql to enable **expanded display** mode, which makes wide rows easier to read:

  ```sql
  \x
  SELECT * FROM weather;
  ```
</Tip>

## Run a transaction

```sql
BEGIN;

UPDATE weather SET temp_hi = temp_hi - 2, temp_lo = temp_lo - 2
WHERE date > '1994-11-28';

-- Check the changes before committing
SELECT * FROM weather;

COMMIT;
```

If something goes wrong, use `ROLLBACK` instead of `COMMIT` to discard the changes.
