title: libpq — C client library
description: The C application programmer's interface to PostgreSQL. Covers connection functions, query execution, result handling, asynchronous queries, COPY, and SSL.
---

libpq is the C application programmer's interface to PostgreSQL. It is a set of library functions that allow client programs to pass queries to the PostgreSQL backend server and to receive the results of those queries.

libpq is also the underlying engine for several other PostgreSQL application interfaces, including those written for C++, Perl, Python, and Tcl.

Client programs that use libpq must include the header file `libpq-fe.h` and must link with the `libpq` library.

```bash
gcc myapp.c -o myapp $(pg_config --cflags) -lpq $(pg_config --ldflags)
```

## Connecting to the database

Each connection is represented by a `PGconn` object. The library always returns a non-null pointer from the connection functions; you must call `PQstatus` to determine whether the connection succeeded.

<Warning>
If untrusted users have access to a database that has not adopted a secure schema usage pattern, begin each session by removing publicly-writable schemas from `search_path`. You can set parameter keyword `options` to value `-csearch_path=`, or issue `PQexec(conn, "SELECT pg_catalog.set_config('search_path', '', false)")` after connecting.
</Warning>

<Warning>
On Unix, forking a process with open libpq connections can lead to unpredictable results because the parent and child processes share the same sockets and operating system resources. Such usage is not recommended, though doing an `exec` from the child process to load a new executable is safe.
</Warning>

### PQconnectdb

Opens a new database connection using a connection string.

```c
PGconn *PQconnectdb(const char *conninfo);
```

The `conninfo` string can be empty to use all default parameters, contain one or more parameter settings separated by whitespace, or be a URI. Examples:

```
host=localhost port=5432 dbname=mydb user=myuser password=secret
postgresql://myuser:secret@localhost:5432/mydb
postgresql://localhost/mydb?user=myuser&sslmode=require
```

### PQconnectdbParams

Opens a connection using two NULL-terminated arrays of keywords and values. Preferred for new application programming because the parameter set can be extended without changing the function signature.

```c
PGconn *PQconnectdbParams(const char * const *keywords,
                          const char * const *values,
                          int expand_dbname);
```

When `expand_dbname` is non-zero, the first `dbname` keyword value is checked to see if it is a connection string, and if so it is expanded into individual parameters.

**Example:**

```c
#include <libpq-fe.h>
#include <stdio.h>
#include <stdlib.h>

int main(void)
{
    const char *keywords[] = { "host", "port", "dbname", "user", NULL };
    const char *values[]   = { "localhost", "5432", "mydb", "myuser", NULL };

    PGconn *conn = PQconnectdbParams(keywords, values, 0);

    if (PQstatus(conn) != CONNECTION_OK) {
        fprintf(stderr, "Connection failed: %s\n", PQerrorMessage(conn));
        PQfinish(conn);
        exit(1);
    }

    printf("Connected to database: %s\n", PQdb(conn));
    PQfinish(conn);
    return 0;
}
```

### Connection string keywords

The following are the most commonly used connection parameter keywords:

| Keyword | Default | Description |
|---------|---------|-------------|
| `host` | localhost | Host name or IP address of the server |
| `hostaddr` | — | Numeric IP address; prevents DNS lookups |
| `port` | 5432 | Port number |
| `dbname` | username | Database name |
| `user` | OS user name | PostgreSQL user name |
| `password` | — | Password |
| `passfile` | `~/.pgpass` | Password file |
| `connect_timeout` | 0 | Maximum wait in seconds (0 = wait indefinitely) |
| `sslmode` | prefer | SSL mode: `disable`, `allow`, `prefer`, `require`, `verify-ca`, `verify-full` |
| `sslcert` | — | Client certificate file |
| `sslkey` | — | Client private key file |
| `sslrootcert` | — | Root certificate file |
| `options` | — | Command-line options to send to the server at connection start |
| `application_name` | — | Specifies `application_name` GUC |
| `target_session_attrs` | any | `read-write`, `read-only`, `primary`, `standby`, `prefer-standby`, `any` |

## Connection status functions

### PQstatus

```c
ConnStatusType PQstatus(const PGconn *conn);
```

Returns the status of the connection. The most important values are `CONNECTION_OK` and `CONNECTION_BAD`. A non-`CONNECTION_OK` status can be recovered from by calling `PQreset`.

### PQerrorMessage

```c
char *PQerrorMessage(const PGconn *conn);
```

Returns the most recent error message generated by an operation on the connection. Nearly all libpq functions set this on failure.

### Other status accessors

| Function | Returns |
|----------|---------|
| `PQdb(conn)` | Database name |
| `PQuser(conn)` | User name |
| `PQhost(conn)` | Server host |
| `PQport(conn)` | Server port |
| `PQserverVersion(conn)` | Server version as integer (e.g., `160001`) |
| `PQsocket(conn)` | File descriptor of the connection socket |
| `PQbackendPID(conn)` | PID of the backend process |
| `PQsslInUse(conn)` | 1 if SSL is in use |

### PQfinish

```c
void PQfinish(const PGconn *conn);
```

Closes the connection to the server and frees memory used by the `PGconn` object. You must call this function when you are done with a connection, even if the connection attempt failed.

### PQreset

```c
void PQreset(PGconn *conn);
```

Resets the communication channel to the server. Useful when a connection is broken.

## Executing queries

### PQexec

The simplest way to execute a command. Waits for the query to complete and returns the result.

```c
PGresult *PQexec(PGconn *conn, const char *command);
```

<Warning>
Use `PQexecParams` instead of `PQexec` when passing user-supplied values to avoid SQL injection. `PQexec` does not support parameterized queries.
</Warning>

### PQexecParams

Executes a command with separately provided parameter values. Parameters are referenced as `$1`, `$2`, etc. in the query text.

```c
PGresult *PQexecParams(PGconn *conn,
                       const char *command,
                       int nParams,
                       const Oid *paramTypes,
                       const char * const *paramValues,
                       const int *paramLengths,
                       const int *paramFormats,
                       int resultFormat);
```

**Example — safe parameterized query:**

```c
const char *paramValues[1];
paramValues[0] = "42";

PGresult *res = PQexecParams(conn,
    "SELECT name, email FROM users WHERE id = $1",
    1,           /* one parameter */
    NULL,        /* let backend deduce param types */
    paramValues,
    NULL,        /* lengths not needed for text params */
    NULL,        /* all params are text format */
    0);          /* request text results */
```

### PQprepare and PQexecPrepared

For queries that will be executed many times, prepare the statement once and execute it repeatedly.

```c
PGresult *PQprepare(PGconn *conn,
                    const char *stmtName,
                    const char *query,
                    int nParams,
                    const Oid *paramTypes);

PGresult *PQexecPrepared(PGconn *conn,
                         const char *stmtName,
                         int nParams,
                         const char * const *paramValues,
                         const int *paramLengths,
                         const int *paramFormats,
                         int resultFormat);
```

**Example:**

```c
/* Prepare once */
PGresult *res = PQprepare(conn, "get_user",
    "SELECT name, email FROM users WHERE id = $1",
    1, NULL);
PQclear(res);

/* Execute multiple times */
for (int i = 1; i <= 100; i++) {
    char id_str[16];
    snprintf(id_str, sizeof(id_str), "%d", i);
    const char *paramValues[1] = { id_str };

    res = PQexecPrepared(conn, "get_user", 1, paramValues, NULL, NULL, 0);
    /* ... process result ... */
    PQclear(res);
}
```

## Handling results

All query functions return a `PGresult *`. You must always call `PQclear` when you are done with a result to avoid memory leaks.

### PQresultStatus

```c
ExecStatusType PQresultStatus(const PGresult *res);
```

Returns the result status. Common values:

| Status | Meaning |
|--------|---------|
| `PGRES_COMMAND_OK` | Command completed successfully, no rows returned |
| `PGRES_TUPLES_OK` | Successful query returning rows |
| `PGRES_COPY_IN` | Server ready for COPY data from client |
| `PGRES_COPY_OUT` | Server is sending COPY data |
| `PGRES_FATAL_ERROR` | Query failed |

### Accessing rows and columns

```c
int    PQntuples(const PGresult *res);           /* number of rows */
int    PQnfields(const PGresult *res);           /* number of columns */
char  *PQfname(const PGresult *res, int col);   /* column name */
int    PQfnumber(const PGresult *res, const char *name); /* column index */
Oid    PQftype(const PGresult *res, int col);   /* column type OID */
char  *PQgetvalue(const PGresult *res, int row, int col); /* cell value */
int    PQgetisnull(const PGresult *res, int row, int col); /* 1 if NULL */
int    PQgetlength(const PGresult *res, int row, int col); /* byte length */
```

### PQclear

```c
void PQclear(PGresult *res);
```

Frees the storage associated with a `PGresult`. Every non-null result must be freed with this function.

**Complete query example:**

```c
PGresult *res = PQexec(conn, "SELECT id, name FROM products ORDER BY id");

if (PQresultStatus(res) != PGRES_TUPLES_OK) {
    fprintf(stderr, "Query failed: %s\n", PQerrorMessage(conn));
    PQclear(res);
    PQfinish(conn);
    exit(1);
}

int rows = PQntuples(res);
int cols = PQnfields(res);

for (int i = 0; i < rows; i++) {
    for (int j = 0; j < cols; j++) {
        printf("%s: %s  ",
            PQfname(res, j),
            PQgetisnull(res, i, j) ? "NULL" : PQgetvalue(res, i, j));
    }
    printf("\n");
}

PQclear(res);
```

## Asynchronous query processing

For non-blocking operation, use the asynchronous API. This is important in event-driven applications.

### PQsendQuery

```c
int PQsendQuery(PGconn *conn, const char *command);
```

Submits a command without waiting for result(s). Returns 1 on success, 0 on failure. After calling `PQsendQuery`, call `PQgetResult` one or more times to get the results.

### PQsendQueryParams and PQsendPrepare

```c
int PQsendQueryParams(PGconn *conn,
                      const char *command,
                      int nParams,
                      const Oid *paramTypes,
                      const char * const *paramValues,
                      const int *paramLengths,
                      const int *paramFormats,
                      int resultFormat);

int PQsendPrepare(PGconn *conn,
                  const char *stmtName,
                  const char *query,
                  int nParams,
                  const Oid *paramTypes);

int PQsendQueryPrepared(PGconn *conn,
                        const char *stmtName,
                        int nParams,
                        const char * const *paramValues,
                        const int *paramLengths,
                        const int *paramFormats,
                        int resultFormat);
```

### PQgetResult

```c
PGresult *PQgetResult(PGconn *conn);
```

Waits for the next result from the last `PQsendQuery` call. Returns NULL when no more results are available. You must call `PQgetResult` and discard all results until NULL is returned before issuing another command.

### PQconsumeInput and PQisBusy

```c
int PQconsumeInput(PGconn *conn);  /* absorb available input from server */
int PQisBusy(PGconn *conn);        /* 1 if a query result is not yet available */
```

Use these in a polling loop to avoid blocking:

```c
/* Send query without blocking */
PQsendQuery(conn, "SELECT pg_sleep(2)");

/* Poll until result available */
while (PQisBusy(conn)) {
    /* do other work, or sleep briefly */
    PQconsumeInput(conn);
}

PGresult *res;
while ((res = PQgetResult(conn)) != NULL) {
    /* process result */
    PQclear(res);
}
```

### Non-blocking connection

To avoid blocking during connection establishment:

```c
PGconn *conn = PQconnectStart(conninfo);
if (!conn) { /* out of memory */ }

if (PQstatus(conn) == CONNECTION_BAD) {
    /* connection failed immediately */
}

/* Poll until connected */
PostgresPollingStatusType poll_status;
do {
    int fd = PQsocket(conn);
    /* use select()/poll() to wait for fd readability/writability */
    poll_status = PQconnectPoll(conn);
} while (poll_status != PGRES_POLLING_OK && poll_status != PGRES_POLLING_FAILED);
```

## COPY protocol

The COPY protocol allows high-speed bulk data transfer between the client and server.

```c
/* Copy data from client to server */
int PQputCopyData(PGconn *conn, const char *buffer, int nbytes);
int PQputCopyEnd(PGconn *conn, const char *errormsg);

/* Copy data from server to client */
int PQgetCopyData(PGconn *conn, char **buffer, int async);
```

**Example — COPY IN:**

```c
PGresult *res = PQexec(conn, "COPY mytable FROM STDIN WITH (FORMAT csv)");
if (PQresultStatus(res) != PGRES_COPY_IN) { /* error */ }
PQclear(res);

const char *row = "1,Alice,alice@example.com\n";
PQputCopyData(conn, row, strlen(row));

/* Signal end of data */
PQputCopyEnd(conn, NULL);

/* Retrieve final status */
res = PQgetResult(conn);
PQclear(res);
```

## SSL connections

TLS/SSL support is configured through connection parameters and can be inspected after connecting.

```c
/* Check if SSL is in use */
int PQsslInUse(const PGconn *conn);

/* Get SSL attribute value */
const char *PQsslAttribute(const PGconn *conn, const char *attribute_name);

/* List available attribute names */
const char * const *PQsslAttributeNames(const PGconn *conn);

/* Get a pointer to the implementation-specific SSL object (e.g., OpenSSL SSL*) */
void *PQsslStruct(const PGconn *conn, const char *struct_name);
```

Common SSL attributes returned by `PQsslAttribute`:

| Attribute | Description |
|-----------|-------------|
| `library` | SSL implementation name (e.g., `"OpenSSL"`) |
| `protocol` | Protocol version (e.g., `"TLSv1.3"`) |
| `key_bits` | Key strength in bits |
| `cipher` | Cipher suite name |
| `compression` | `"on"` or `"off"` |

**Example — connecting with SSL and verifying:**

```c
const char *keywords[] = {
    "host", "dbname", "sslmode", "sslrootcert", NULL
};
const char *values[] = {
    "db.example.com", "mydb", "verify-full", "/etc/ssl/certs/ca.crt", NULL
};

PGconn *conn = PQconnectdbParams(keywords, values, 0);

if (PQstatus(conn) == CONNECTION_OK && PQsslInUse(conn)) {
    printf("SSL: %s / %s\n",
        PQsslAttribute(conn, "protocol"),
        PQsslAttribute(conn, "cipher"));
}
```

## Environment variables

When a connection parameter is not supplied in the connection string or keyword arrays, libpq looks for the following environment variables:

| Variable | Keyword |
|----------|----------|
| `PGHOST` | `host` |
| `PGPORT` | `port` |
| `PGDATABASE` | `dbname` |
| `PGUSER` | `user` |
| `PGPASSWORD` | `password` |
| `PGPASSFILE` | `passfile` |
| `PGSSLMODE` | `sslmode` |
| `PGCONNECT_TIMEOUT` | `connect_timeout` |
| `PGAPPNAME` | `application_name` |

## Complete example

```c
#include <stdio.h>
#include <stdlib.h>
#include <libpq-fe.h>

static void exit_nicely(PGconn *conn)
{
    PQfinish(conn);
    exit(1);
}

int main(void)
{
    PGconn   *conn;
    PGresult *res;

    conn = PQconnectdb("host=localhost dbname=testdb");

    if (PQstatus(conn) != CONNECTION_OK) {
        fprintf(stderr, "Connection failed: %s\n", PQerrorMessage(conn));
        exit_nicely(conn);
    }

    /* Parameterized query — safe against SQL injection */
    const char *paramValues[1] = { "100" };
    res = PQexecParams(conn,
                       "SELECT id, name FROM orders WHERE total > $1",
                       1, NULL, paramValues, NULL, NULL, 0);

    if (PQresultStatus(res) != PGRES_TUPLES_OK) {
        fprintf(stderr, "Query failed: %s\n", PQerrorMessage(conn));
        PQclear(res);
        exit_nicely(conn);
    }

    int rows = PQntuples(res);
    printf("%d rows:\n", rows);
    for (int i = 0; i < rows; i++)
        printf("  %s  %s\n",
            PQgetvalue(res, i, 0),
            PQgetvalue(res, i, 1));

    PQclear(res);
    PQfinish(conn);
    return 0;
}
```
